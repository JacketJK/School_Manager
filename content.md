# Class Room Manager | ระบบบริหารห้องเรียน

ระบบจัดการห้องเรียนสำหรับนักเรียน ใช้ติดตามการบ้าน คะแนนเก็บรายวิชา และภาพรวมผลการเรียน ทำงานเป็นเว็บแอปฝั่ง Frontend ล้วน (ไม่ต้องมีเซิร์ฟเวอร์หรือฐานข้อมูล) และบันทึกข้อมูลไว้ใน `localStorage` ของเบราว์เซอร์ผู้ใช้แต่ละคน

---

## 1. ภาพรวมระบบ

หน้าจอหลักแบ่งออกเป็น 4 ส่วน:

| ส่วน | ตำแหน่ง | หน้าที่ |
|---|---|---|
| **การบ้านของฉัน** | ซ้าย | รายการการบ้าน แยกตาม วันนี้ / พรุ่งนี้ และกรองตามวิชาได้ |
| **เวลาเรียน / ผลการเรียน** | กลางบน | กราฟวงกลมแสดง % การบ้านที่ทำเสร็จ และกราฟพื้นที่เปรียบเทียบคะแนนแต่ละวิชา 2 ภาคเรียน |
| **คะแนนเก็บรายวิชา** | กลางล่าง | แถบคะแนนของแต่ละวิชา แก้ไขได้ |
| **กิจกรรมโรงเรียน / ประชาสัมพันธ์** | ขวา | ประกาศและกิจกรรมของโรงเรียน (ปัจจุบันเป็นข้อมูลตัวอย่างแบบ static) |

## 2. โครงสร้างไฟล์

```
School_Manager/
├── index.html   โครงหน้าเว็บ + modal สำหรับเพิ่ม/แก้ไขข้อมูล
├── styles.css   ดีไซน์และ layout ทั้งหมด
├── script.js    ตรรกะของระบบ: จัดการข้อมูล, render UI, กราฟ, modal
└── content.md   เอกสารนี้
```

ไม่มีขั้นตอน build หรือ dependency ที่ต้องติดตั้ง เปิด `index.html` ด้วยเบราว์เซอร์ได้ทันที (ไลบรารีภายนอกอย่าง Bootstrap Icons และ ApexCharts โหลดผ่าน CDN)

## 3. ฟีเจอร์ที่ใช้งานได้จริง

### 3.1 การบ้าน (Homework)
- **เพิ่ม**: กดปุ่ม `+` มุมขวาบนของกล่อง "การบ้านของฉัน" → กรอกวิชา / ชื่องาน / รายละเอียด / กำหนดส่ง (วันนี้หรือพรุ่งนี้) → บันทึก
- **แก้ไข**: คลิกที่การ์ดการบ้านเพื่อเปิดฟอร์มแก้ไขข้อมูลเดิม
- **ลบ**: เปิดฟอร์มแก้ไข แล้วกดปุ่ม "ลบ"
- **ทำเครื่องหมายว่าทำเสร็จ**: คลิกไอคอนวงกลม (✓) ที่มุมขวาบนของการ์ด การ์ดจะจางลงและมีขีดทับชื่องาน
- **กรองตามวิชา**: เลือกวิชาจาก dropdown เหนือรายการการบ้าน (ดึงรายชื่อวิชาจริงจากระบบ ไม่ได้ล็อกไว้ตายตัว)
- **ค้นหา**: ช่องค้นหาด้านบนของหน้า ค้นจากชื่องาน รายละเอียด และชื่อวิชา แบบเรียลไทม์

### 3.2 คะแนนเก็บรายวิชา (Grades)
- กดไอคอนรูป sliders ที่หัวข้อ "คะแนนเก็บรายวิชา" เพื่อเปิดหน้าต่างแก้ไข
- แก้ไขคะแนนภาคเรียนที่ 1 และภาคเรียนที่ 2 ของแต่ละวิชา (0–100)
- เพิ่มวิชาใหม่ หรือลบวิชาที่ไม่ต้องการออกจากระบบได้ในหน้าต่างเดียวกัน
- เมื่อบันทึก แถบคะแนนและกราฟ "ผลการเรียน" จะอัปเดตทันที

### 3.3 กราฟและสรุปผล
- **เวลาเรียน (กราฟวงกลม)**: คำนวณจากสัดส่วนการบ้านทั้งหมดที่ถูกทำเครื่องหมายว่าเสร็จแล้ว
- **ผลการเรียน (กราฟพื้นที่)**: เปรียบเทียบคะแนนภาคเรียนที่ 1 กับภาคเรียนที่ 2 ของทุกวิชาที่มีในระบบ

### 3.4 อื่น ๆ ที่มีอยู่เดิม
- เปิด/ปิดเมนูด้านข้าง (sidebar)
- เมนู dropdown โปรไฟล์ผู้ใช้
- ปุ่มสลับช่วงเวลา (วันนี้ / สัปดาห์นี้ / เดือนนี้ / รายงาน) — ปัจจุบันเปลี่ยนสถานะ active เท่านั้น ยังไม่ผูกกับการกรองข้อมูลจริง
- นาฬิกา/วันที่แบบเรียลไทม์ (ภาษาไทย)

## 4. การเก็บข้อมูล (Data Model)

ข้อมูลทั้งหมดเก็บเป็น JSON ก้อนเดียวใน `localStorage` ภายใต้คีย์ `crm_data_v1` โครงสร้างข้อมูล:

```js
{
  subjects: ["ภาษาไทย", "คณิตศาสตร์", ...],      // รายชื่อวิชาทั้งหมด
  grades: {
    "ภาษาไทย": { previous: 66, current: 50 },     // คะแนนภาคเรียนก่อนหน้า/ปัจจุบัน
    ...
  },
  homework: [
    {
      id: "hw1",
      subject: "คณิตศาสตร์",
      title: "...",
      detail: "...",
      due: "today" | "tomorrow",
      done: false
    },
    ...
  ]
}
```

**ข้อควรทราบ**
- ข้อมูลผูกกับเบราว์เซอร์/เครื่องที่ใช้งานเท่านั้น ไม่ sync ข้ามอุปกรณ์ และจะหายไปถ้าล้างข้อมูลเบราว์เซอร์ (Clear browsing data)
- ถ้าต้องการรีเซ็ตข้อมูลกลับเป็นค่าเริ่มต้น ให้เปิด Developer Console แล้วรัน `localStorage.removeItem('crm_data_v1')` จากนั้นรีเฟรชหน้า

## 5. สิ่งที่ยังไม่รองรับ (ข้อจำกัดปัจจุบัน)

- ไม่มีระบบล็อกอิน/ผู้ใช้จริง (ข้อมูลโปรไฟล์ด้านบนยังเป็นข้อความคงที่)
- กิจกรรมโรงเรียนและประชาสัมพันธ์ยังเป็นข้อมูลตัวอย่าง แก้ไข/เพิ่มไม่ได้
- ไม่มีฐานข้อมูลกลาง จึงไม่เหมาะกับการใช้งานหลายคนพร้อมกันหรือดูข้อมูลข้ามอุปกรณ์
- ปุ่มช่วงเวลา (วันนี้/สัปดาห์นี้/เดือนนี้/รายงาน) ยังไม่กรองข้อมูลจริง

## 6. แนวทางพัฒนาต่อ (แนะนำ)

- เพิ่ม backend + ฐานข้อมูลจริง (เช่น Firebase, Supabase หรือ REST API) เพื่อ sync ข้ามอุปกรณ์และรองรับผู้ใช้หลายคน
- เพิ่มระบบล็อกอิน
- ทำให้กิจกรรม/ประกาศแก้ไขได้เหมือนการบ้าน
- ผูกปุ่มช่วงเวลาด้านบนเข้ากับการกรองการบ้าน/กิจกรรมตามวันจริง
- เพิ่มระบบสิทธิ์การใช้งานตามบทบาท (ดูรายละเอียดในหัวข้อ 7)

## 7. ระบบสิทธิ์การใช้งานตามบทบาท (Role-Based Access)

> **สถานะ**: เป็นสเปกออกแบบสำหรับพัฒนาต่อ ระบบปัจจุบันยังไม่มีการล็อกอิน/แยกผู้ใช้จริง (ดูข้อจำกัดในหัวข้อ 5) การนำไปใช้งานจริงต้องเพิ่มระบบยืนยันตัวตน (authentication) และ backend เก็บบัญชีผู้ใช้ก่อน

ระบบจะรองรับผู้ใช้งาน 6 บทบาท แต่ละบทบาทมองเห็นและแก้ไขข้อมูลได้ไม่เท่ากัน เพื่อให้ข้อมูลนักเรียนถูกดูแลโดยผู้ที่เกี่ยวข้องเท่านั้น

### 7.1 บทบาททั้งหมด

| บทบาท | ขอบเขตความรับผิดชอบ |
|---|---|
| **นักเรียน** (Student) | เจ้าของข้อมูล ดูและจัดการการบ้าน/ตารางเรียนของตนเอง |
| **ครูประจำวิชา** (Subject Teacher) | ดูแลเนื้อหา การบ้าน และคะแนนเฉพาะวิชาที่ตนสอน ในทุกห้องที่รับผิดชอบ |
| **ครูประจำชั้น** (Homeroom Teacher) | ดูแลภาพรวมของนักเรียนในห้องที่ตนดูแล ทุกวิชา รวมถึงพฤติกรรมและการแจ้งเตือนผู้ปกครอง |
| **ฝ่ายวิชาการ** (Academic Affairs) | ดูแลโครงสร้างวิชา หลักสูตร ตารางสอบ และภาพรวมผลการเรียนทั้งโรงเรียน |
| **ฝ่ายปกครอง** (Student Affairs / Discipline) | ดูแลความประพฤติ การเข้าร่วมกิจกรรม การมาเรียน และประกาศด้านระเบียบวินัย |
| **แอดมิน** (Admin) | สิทธิ์สูงสุด จัดการบัญชีผู้ใช้ กำหนดบทบาท และตั้งค่าระบบทั้งหมด |

### 7.2 ตารางสิทธิ์การใช้งาน (Permission Matrix)

สัญลักษณ์: ✅ ทำได้เต็มสิทธิ์ · 👁 ดูได้อย่างเดียว · 🔒 เฉพาะขอบเขตของตน (วิชา/ห้อง/นักเรียนที่รับผิดชอบ) · ❌ ไม่มีสิทธิ์

| การทำงาน | นักเรียน | ครูประจำวิชา | ครูประจำชั้น | ฝ่ายวิชาการ | ฝ่ายปกครอง | แอดมิน |
|---|---|---|---|---|---|---|
| เพิ่ม/แก้ไข/ลบการบ้านของตนเอง | ✅ | – | – | – | – | ✅ |
| เพิ่ม/แก้ไข/ลบการบ้านให้ทั้งห้อง (วิชาที่สอน) | ❌ | 🔒 | 👁 | 👁 | ❌ | ✅ |
| กรอก/แก้ไขคะแนนเก็บ (วิชาที่สอน) | ❌ | 🔒 | 👁 | 👁 | ❌ | ✅ |
| ดูคะแนน/ผลการเรียนทุกวิชาของห้องที่ดูแล | 👁 (ของตน) | 🔒 (เฉพาะวิชาตน) | 🔒 (ห้องตน) | ✅ (ทุกห้อง) | 👁 | ✅ |
| จัดการรายชื่อวิชา/หลักสูตร | ❌ | ❌ | ❌ | ✅ | ❌ | ✅ |
| บันทึกพฤติกรรม/การมาเรียน | ❌ | ❌ | 🔒 (ห้องตน) | 👁 | ✅ | ✅ |
| สร้าง/แก้ไขกิจกรรมโรงเรียน | ❌ | 🔒 (กิจกรรมวิชาตน) | 🔒 (กิจกรรมห้องตน) | 👁 | ✅ | ✅ |
| สร้าง/แก้ไขประกาศ | ❌ | 🔒 (ประกาศวิชาตน) | 🔒 (ประกาศห้องตน) | ✅ (ระดับโรงเรียน) | ✅ (ระเบียบวินัย) | ✅ |
| ออกรายงานสรุปผลการเรียนทั้งโรงเรียน | ❌ | ❌ | 👁 (ห้องตน) | ✅ | 👁 | ✅ |
| จัดการบัญชีผู้ใช้และกำหนดบทบาท | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ |

### 7.3 มุมมองหน้าจอที่ต่างกันตามบทบาท

- **นักเรียน**: เห็นหน้าปัจจุบันตามที่ใช้งานอยู่ตอนนี้ (การบ้าน/คะแนน/กราฟของตัวเอง)
- **ครูประจำวิชา**: แทนกล่อง "การบ้านของฉัน" ด้วยรายการ "การบ้านที่มอบหมาย" แยกตามห้อง และแก้ไขคะแนนได้เฉพาะวิชาที่สอน
- **ครูประจำชั้น**: เพิ่มแดชบอร์ดภาพรวมนักเรียนทั้งห้อง (สรุปคะแนนเฉลี่ย/การบ้านค้าง/พฤติกรรม) แทนมุมมองรายบุคคล
- **ฝ่ายวิชาการ**: เพิ่มเมนูจัดการวิชา/หลักสูตร และรายงานเปรียบเทียบผลการเรียนระหว่างห้อง/ระดับชั้น
- **ฝ่ายปกครอง**: เพิ่มเมนูบันทึกพฤติกรรมและการมาเรียน พร้อมประกาศด้านระเบียบวินัย
- **แอดมิน**: เพิ่มเมนู "จัดการผู้ใช้" สำหรับสร้างบัญชี กำหนด/เปลี่ยนบทบาท และรีเซ็ตรหัสผ่าน

### 7.4 การเปลี่ยนแปลงโครงสร้างข้อมูลที่ต้องเพิ่ม

เพื่อรองรับบทบาทเหล่านี้ โครงสร้างข้อมูลในหัวข้อ 4 ต้องขยายเพิ่ม เช่น:

```js
{
  users: [
    {
      id: "u1",
      name: "ครูสมชาย ใจดี",
      role: "subject_teacher",      // student | subject_teacher | homeroom_teacher
                                     // academic_affairs | discipline_affairs | admin
      subjects: ["คณิตศาสตร์"],     // ใช้กับ role ที่ผูกกับวิชา
      classRooms: ["ป.6/1"]         // ใช้กับ role ที่ผูกกับห้อง
    }
  ],
  classRooms: [
    { id: "c1", name: "ป.6/1", homeroomTeacherId: "u2", studentIds: ["s1", "s2"] }
  ],
  // homework, grades เดิม จะต้องผูกกับ classRoomId และ createdByUserId เพิ่มเติม
}
```

และฝั่งโค้ดต้องเพิ่ม **ระบบยืนยันตัวตน (login)** และ **ตรวจสอบสิทธิ์ (permission check)** ก่อนแสดงปุ่ม/เมนู หรือก่อนบันทึกข้อมูลทุกจุดที่เกี่ยวข้อง ตามตารางในหัวข้อ 7.2
